Мощный инструмент для веб-сканирования, который интегрируется с ИИ-ассистентами через MCP (Model Context Protocol). Этот проект позволяет ИИ-ассистентам сканировать веб-сайты, извлекать динамический контент, переходить по ссылкам и напрямую сохранять структурированные файлы в формате Markdown.

## 📋 Возможности

- Нативная интеграция с ИИ-ассистентами через MCP
- Возврат извлечённого контента в формате Markdown напрямую в ИИ
- Извлечение и отображение внутренних/внешних ссылок для навигации ИИ
- Сканирование веб-сайтов с настраиваемой глубиной
- Подробная статистика результатов сканирования
- Обработка ошибок и ненайденных страниц
- **Расширенные возможности сканирования**:
  - **Magic Mode**: обход анти-ботов (таких как Cloudflare) и имитация поведения реального браузера
  - **Целевое извлечение**: получение только необходимых данных с помощью CSS-селекторов
  - **Пользовательский JavaScript**: выполнение кода перед извлечением (клики, прокрутка, заполнение форм)
  - **Постоянные сессии**: сохранение файлов cookie и состояния между запросами для сайтов с аутентификацией
  - **Поддержка SPA**: ожидание динамических CSS-селекторов или установка явных задержек перед извлечением

## 🚀 Настройка MCP

Самый простой и рекомендуемый способ использования этого инструмента — через `uvx`, который автоматически загружает и запускает последнюю опубликованную версию с PyPI.

### Предварительные требования

- [uv](https://github.com/astral-sh/uv), установленный в вашей системе.

### Настройка для ИИ-ассистентов (например, Claude Desktop, Cline)

Добавьте следующее в файл конфигурации MCP вашего ИИ-ассистента (например, `cline_mcp_settings.json` или `claude_desktop_config.json`):

> **Примечание для пользователей Windows**: Настоятельно рекомендуется указать `--python 3.13`, чтобы избежать проблем с компиляцией некоторых зависимостей.

```json
{
  "mcpServers": {
    "crawl": {
      "command": "uvx",
      "args": [
        "--python",
        "3.13",
        "crawl4ai-mcp-llm"
      ],
      "disabled": false,
      "autoApprove": [],
      "timeout": 600
    }
  }
}
```

### Важно: Установка браузера

Скрапер использует Playwright для обработки динамического контента. После настройки инструмента необходимо установить требуемые браузеры:

```bash
uv run playwright install chromium
```

## 🖥️ Использование

После настройки вы можете использовать скрапер, попросив вашего ИИ-ассистента выполнить сканирование.

### Примеры использования с Claude/Cline

- **Простое сканирование**: «Можешь просканировать сайт example.com и сделать краткое резюме?»
- **Сканирование с параметрами**: «Можешь просканировать https://example.com с глубиной 3 и включить внешние ссылки?»
- **Динамический контент**: «Просканируй это React-приложение и дождись загрузки селектора `.main-content`.»
- **Обход защиты**: «Просканируй example.com, но используй "magic mode" для обхода анти-бот-защиты.»
- **Целевое извлечение**: «Просканируй сайт документации, но извлекай только контент, соответствующий CSS-селектору `h1, p.lead`.»

## 🛠️ Доступные параметры (инструмент MCP)

Инструмент `crawl` принимает следующие параметры:

| Параметр | Тип | Описание | Значение по умолчанию |
|-----------|------|-------------|---------------|
| `url` | string | URL для сканирования (обязательный) | - |
| `max_depth` | integer | Максимальная глубина сканирования | 2 |
| `include_external` | boolean | Включать внешние ссылки | false |
| `verbose` | boolean | Включить подробный вывод | true |
| `wait_for_selector` | string | CSS-селектор, которого нужно дождаться перед извлечением контента. Полезно для одностраничных приложений. | None |
| `return_content` | boolean | Возвращать ли извлечённый контент напрямую в ответе MCP (при необходимости обрезается до 50 000 символов). | true |
| `output_file` | string | Путь к файлу вывода | генерируется автоматически |
| `magic` | boolean | Включить magic mode для обхода анти-ботов и имитации реального браузера | false |
| `css_selector` | string | Конкретный CSS-селектор для извлечения только целевых элементов со страницы | None |
| `js_code` | string | Пользовательский JavaScript-код для выполнения на странице перед извлечением | None |
| `session_id` | string | Идентификатор постоянной сессии для сохранения файлов cookie и состояния браузера между запросами | None |
| `delay_before_return_html` | number | Задержка в секундах перед извлечением HTML (полезно для страниц с тяжёлым JS) | None |
